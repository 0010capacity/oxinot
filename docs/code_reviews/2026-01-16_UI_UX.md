### 1. 재사용성 (Reusability) & 구조

**현황:**
- **Mantine UI 라이브러리**를 적극 활용하여 기본적인 UI 일관성을 확보했습니다.
- **Store 의존성:** `Breadcrumb.tsx` 등의 컴포넌트가 `useViewStore`, `useBlockStore` 등 전역 상태(Zustand)에 직접 연결되어 있습니다. 이는 해당 앱 내에서는 편리하지만, 컴포넌트 단위 테스트나 다른 컨텍스트에서의 재사용을 어렵게 만듭니다.
- **Monolithic Component:** `SettingsModal.tsx` 파일이 약 800라인에 달하며, 모든 탭의 UI와 로직, 검색 기능이 한 파일에 뭉쳐 있습니다.

**개선 제안:**
- **컴포넌트 분리:** `SettingsModal` 내부의 각 탭(Appearance, Git, Advanced 등)을 별도 컴포넌트로 분리해야 합니다.
  ```tsx
  // 추천 구조
  src/components/settings/
  ├── SettingsModal.tsx (Layout & Search 로직 담당)
  ├── SettingsAppearance.tsx
  ├── SettingsGit.tsx
  └── ...
  ```
- **Presentational Component 분리:** `Breadcrumb`에서 데이터를 가져오는 로직(Container)과 렌더링하는 로직(Presenter)을 분리하면 테스트와 관리가 용이해집니다.

### 2. 스타일링 (Styling)

**현황:**
- **CSS Variables:** `src/styles/variables.css`에 디자인 토큰(`--color-bg-primary` 등)이 잘 정의되어 있고, 다크 모드 대응이 용이한 구조입니다.
- **인라인 스타일 과용:** `CollapseToggle.tsx`에서 Hover 효과를 CSS(`:hover`)가 아닌 JS 이벤트(`onMouseEnter`, `onMouseLeave`)와 인라인 스타일로 처리하고 있습니다. 이는 불필요한 리렌더링을 유발하고 스타일 관리를 어렵게 합니다.

**개선 제안:**
- **CSS 클래스 활용:** `CollapseToggle`의 동적 스타일링을 CSS 파일로 옮기세요.
  ```css
  /* src/components/common/common.css (예시) */
  .collapse-toggle:hover {
    opacity: 1;
    background-color: var(--color-interactive-hover);
  }
  ```

### 3. 국제화 (i18n)

**현황:**
- `react-i18next`가 잘 설정되어 있으며(`src/i18n.ts`), 대부분의 주요 UI 텍스트는 `t()` 함수를 사용 중입니다.
- **하드코딩된 문자열 발견:**
  - `Breadcrumb.tsx`: `"Untitled Block"`
  - `SettingsModal.tsx`: 폰트명("System Default"), 타임존 설정 로직, 날짜 포맷 예시 등에서 하드코딩된 영문 텍스트 존재.
  - `CollapseToggle.tsx`: `aria-label`에 "Expand", "Collapse"가 하드코딩됨.

**개선 제안:**
- 발견된 하드코딩 문자열을 `ko.json`, `en.json`으로 추출하고 키로 대체하세요.
  ```tsx
  // CollapseToggle.tsx
  aria-label={isCollapsed ? t('common.expand') : t('common.collapse')}
  ```

### 4. 접근성 (Accessibility)

**현황:**
- **Mantine 활용:** Modal, Input 등의 폼 요소는 라이브러리 차원에서 접근성이 준수합니다.
- **Semantic HTML 위반:**
  - `Breadcrumb.tsx`: 클릭 가능한 경로 요소들이 `<button>`이나 `<a>` 태그가 아닌 `<Text>`(div/span)에 `onClick` 이벤트를 달아 구현되었습니다. 이는 키보드 사용자(Tab 키 이동 불가)와 스크린 리더 사용자에게 장벽이 됩니다.
- **`CollapseToggle.tsx`:** `<button>` 태그와 `aria-label`을 사용하여 접근성이 우수합니다.

**개선 제안:**
- **Interactive Element 교체:** `Breadcrumb`의 클릭 가능한 요소를 `<button type="button">`으로 감싸거나 교체하고, 스타일을 초기화하여 텍스트처럼 보이게 만드세요.
  ```tsx
  // Breadcrumb.tsx 수정 예시
  <button
    type="button"
    onClick={...}
    className="breadcrumb-item" // 스타일로 button 모양 제거
    style={{ cursor: 'pointer', /* ... */ }}
  >
    <Text>{truncateText(item)}</Text>
  </button>
  ```
