# UI/UX 개선사항 제안

**프로젝트**: Oxinot (Modern Markdown Outliner)  
**작성일**: 2025-01-07  
**버전**: 1.0.0  
**상태**: 제안 단계

---

## 📋 개요

본 문서는 Oxinot 애플리케이션의 사용자 경험(UX)과 사용자 인터페이스(UI)를 개선하기 위한 제안사항을 체계적으로 정리한 것입니다. 현재 구현된 기능을 바탕으로 우선순위별로 구현 가능한 개선안을 제시합니다.

### 목표
- 사용자 경험 최적화
- 접근성 향상
- 시각적 피드백 강화
- 워크플로우 효율성 개선

---

## 🎯 개선사항 카테고리

### 1. 사용자 경험 (UX) 개선

#### 1.1 로딩 상태 개선
**문제점**
- 페이지/블록 로드 시 시각적 피드백 부족
- 대용량 파일 처리 시 진행 상황 미확인
- 로딩 중에도 UI 반응 필요

**개선안**
- 스켈레톤 로더 추가 (페이지 로드 시)
- 진행률 표시기 (대용량 파일 처리 시)
- 로딩 상태에서도 UI 반응 유지

**우선순위**: 높음

---

#### 1.2 온보딩 개선
**문제점**
- 첫 사용자에게 안내가 부족
- 키보드 단축키 가이드 없음
- 샘플 템플릿 미제공

**개선안**
- 인터랙티브 투어 추가 (Tab/Shift+Tab 설명 등)
- 키보드 단축키 가이드 모달 (Cmd/Ctrl+/)
- 샘플 템플릿 제공 (TODO, 프로젝트 관리 등)
- 사용자 매뉴얼 내장

**우선순위**: 중간

---

### 2. 사용자 인터페이스 (UI) 개선

#### 2.1 브레드크럼 개선
**문제점**
- 긴 경로의 경우 텍스트가 너무 많이 잘림
- 현재 경로와 상위 경로 구분이 명확하지 않음
- 드롭다운 메뉴 없음

**개선안**
- 드롭다운 메뉴로 숨겨진 경로 표시
- 호버 시 툴팁으로 전체 경로 표시
- 경로 아이콘 추가 (폴더, 파일 등)
- 클릭 가능한 경로 명확화

**우선순위**: 높음

---

#### 2.2 타이틀바 최적화
**문제점**
- 네비게이션 버튼이 활성/비활성 상태 시 명확하지 않음
- 툴팁 부족
- 버튼 상호작션 피드백 부족

**개선안**
- 비활성 버튼 투명도 조정 (`--opacity-disabled` 활용)
- 툴팁 추가 (호버 시 동작 설명)
- 뒤로가기/앞으로가기 이력 표시 (소규모)
- 버튼 클릭 리플 효과

**우선순위**: 중간

---

#### 2.3 블록 인터랙션 개선
**문제점**
- 블록 선택 시 시각적 피드백이 부족
- 드래그 앤 드롭 시 플레이스홀더 없음
- 블록 삭제 시 애니메이션 없음

**개선안**
- 선택된 블록 배경색 강화 (`var(--color-interactive-selected)`)
- 블록 드래그 시 플레이스홀더 추가
- 블록 삭제 시 애니메이션 추가 (fade-out)
- 블록 접기/펼치기 부드러운 애니메이션

**우선순위**: 높음

---

### 3. 접근성 (Accessibility) 개선

#### 3.1 키보드 네비게이션
**개선안**
- 탭 순서 최적화 (논리적 흐름)
- 포커스 인디케이터 명확화 (CSS 변수 활용)
- 스크린 리더 지원 (ARIA 라벨 추가)
- 키보드 단축키 일관성 유지

**우선순위**: 높음

---

#### 3.2 대비도 개선
**현재 상태**
- CSS 변수로 관리 중 (`--color-bg-primary`, `--color-text-primary` 등)

**개선안**
- WCAG AA 준수 확인 (최소 4.5:1)
- 색맹 모드 지원 (색상 패턴 추가)
- 다크/라이트 모드 전환 시 대비도 유지

**우선순위**: 중간

---

### 4. 에디터 관련 개선

#### 4.1 IME (한글 입력) 개선
**현재 상태**
- `Editor.tsx`에 IME 처리 구현됨
- `compositionstart`, `compositionend` 이벤트 처리

**개선안**
- 한글 입력 중 커서 위치 시각적 피드백
- IME 조합 완료 시 애니메이션 (fade-in)
- IME 조합 중에도 블록 이동 가능

**우선순위**: 높음

---

#### 4.2 마크다운 렌더링
**개선안**
- 라이브 미리보기 모드 (좌우 분할)
- 코드 블록 구문 강조 개선 (현재 `customHighlightStyle` 사용)
- 인라인 코드 백틱 표시 개선
- 테이블 렌더링 지원

**우선순위**: 중간

---

#### 4.3 블록 레벨 UI
**개선안**
- 들여쓰기 가이드 라인 시각화 (현재 `--color-indent-guide`)
- 중첩 레벨별 색상 구분 (선택적)
- 접기/펼치기 인디케이터 개선
- 블록 참조 시각적 표시

**우선순위**: 중간

---

### 5. 검색 및 네비게이션

#### 5.1 검색 모달 개선
**현재 상태**
- `CommandPalette.tsx`, `SearchModal.tsx` 구현됨

**개선안**

- 키보드 단축키 (↑↓로 이동, Enter로 열기)

**우선순위**: 높음

- 검색 하이라이트 (그래프 내 노드)
- 그래프 레이아웃 옵션 (force-directed, radial 등)

## 📝 참고사항

### 기술 제약사항
- **테마 시스템**: 모든 색상은 CSS 변수(`var(--color-*)`)를 사용해야 함
- **CodeMirror**: `defaultHighlightStyle` 대신 `customHighlightStyle`를 사용할 것
- **스타일**: 인라인 스타일 대신 CSS 변수를 활용할 것
- **TypeScript**: `any` 타입 사용 금지, `unknown` 또는 적절한 타입 사용

### 디자인 원칙
- **최소주의**: 불필요한 요소 제거, 중요한 정보에 집중
- **일관성**: 애플리케이션 전체에서 일관된 UI 패턴 유지
- **접근성**: WCAG AA 준수, 키보드 네비게이션 지원
- **성능**: 60fps 유지, 불필요한 리렌더링 방지

### 테스트 계획
- **단위 테스트**: Vitest로 주요 컴포넌트 테스트
- **통합 테스트**: E2E 테스트 (추가 고려)
- **접근성 테스트**: axe-devtools로 접근성 검사
- **성능 테스트**: Lighthouse로 성능 측정
